Docker 镜像通过其分层（layered）架构实现了增量的修改和维护，使得镜像构建、存储和传输变得更加高效和灵活。这一架构依赖于 Union File System（UnionFS）技术，它允许将多个文件系统层叠加在一起，形成一个统一的文件系统视图。在 Docker 的应用中，这种技术不仅优化了数据的存储和共享，还极大地简化了镜像更新和版本控制的过程。

## 1、Docker 镜像的分层原理

Docker 镜像的每一层都代表着镜像构建过程中的一组变更。这些层遵循以下原则：

1. **基础层**：每个镜像的构建都从一个基础层开始，这通常是一个最小化的操作系统镜像。
2. **指令层**：Dockerfile 中的每个指令（如 `RUN`, `COPY`, `ADD`）会在基础层上叠加新的层。
3. **只读层和可写层**：所有这些层在创建时是只读的。容器启动时，在这些只读层顶部添加一个可写层，用于存储运行中的变更。
4. **层的共享和重用**：基于相同基础镜像的不同镜像会共享未变更的层，从而减少存储空间的占用和加速镜像的下载。

## 2、UnionFS 的角色

UnionFS 是实现 Docker 镜像分层的关键技术，它支持将只读层和一个可写层联合在一起。这种结构不仅用于容器化技术中，也被应用于如 Live CD/USB 系统，允许用户在只读系统上执行写操作。

UnionFS 的主要优势包括：

- **数据共享和重用**：通过层的共享，多个容器可以重用相同的数据，减少了冗余数据的存储。
- **即时更新**：用户对容器的修改立即可见，无需更改底层的只读数据。
- **快速部署**：由于大量基础数据的共享，容器可以快速启动，只加载变化的部分。

## 3、增量构建的示例

考虑一个基于 Ubuntu 的 Docker 镜像，其中安装了 Nginx。若后续决定添加 curl 包，仅需创建包含 curl 安装的新层，而 Ubuntu 和 Nginx 层则被重用，不需重新拉取或构建。
